### What Went Well

- The story sequence was staged in thin layers: shell, chronology, comparison, evidence.
- The authored data file stayed the source of truth for narrative, chronology, comparison, and evidence.
- Accessibility stayed central: real buttons, visible focus, keyboard reachability, reduced-motion safety, and no horizontal overflow were treated as product requirements instead of afterthoughts.
- Review feedback improved the work by forcing explicit interaction contracts before the module could grow.
- The module stayed local to the dossier feature; no route, global store, or shared references abstraction was introduced.

### Challenges and Growth Areas

- Selection and focus behavior needed very explicit semantics before the module could grow safely.
- Responsive behavior had to be pinned to concrete breakpoints instead of loose "desktop/mobile" language.
- The evidence surface needed a local disclosure model that reinforced the current context instead of becoming a detached bibliography panel.
- Each new layer had to preserve the recap handoff and canonical anchor behavior, so the reading order never drifted.

### Key Insights

- Shell first, then chronology, then comparison, then evidence is the right decomposition for this chapter.
- The more tightly the data file models the story, the less the UI needs to improvise.
- Context-linked evidence is more trustworthy than a generic references surface.
- Thin local state was enough for the dossier because the boundaries stayed crisp.
