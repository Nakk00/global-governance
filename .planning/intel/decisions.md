# Ingested Decisions

- Student / Expert depth remains a public-shell and chat-context concern, not a global-store or router rewrite.
- Visible source support, clearer weak-support/refusal/fallback states, and better source labels belong to the trust phase and support the current learner flow.
- Maintainer readiness, source status, validation, audit, and error-state clarity remain private stewardship concerns.
- Demo reliability and repeatable verification remain phase-level hygiene, not a new product boundary.
- The archived proposal does not introduce a new runtime boundary or milestone; it sharpens the current brownfield priorities.
