# Ingested Requirements

- Depth-aware chat should also support current lesson context and suggested questions.
- Learners should see visible source support labels or chips, not just hidden citation plumbing.
- Fallback states should be understandable and clearly distinct from transport failure.
- Maintainers should have a readability-first view of source status, validation outcomes, audit entries, and error states.
- The project should keep one documented, fixed-order pre-demo verification path across frontend, functions, backend, and browser checks.
